# Delivery zone resolution as shared workstream

Delivery-zone inference will be treated as a shared checkout workstream rather than a Messenger-only implementation detail. The Messenger bot may use a resolver to suggest top candidates, but customers must confirm until a separate evaluation proves confidence thresholds. The eventual resolver can combine Google Maps/Places normalization, static Ulaanbaatar landmark knowledge, historical order examples, and model ranking so web checkout and Messenger checkout can both hide manual delivery-zone selection when accuracy is proven.
